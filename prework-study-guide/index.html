HTML

  • The head element contains information about the webpage
  • The body element represents the visible content shown to the user.
  • The footer element contains information shown at the bottom of the webpage, normally copyright, author, or other similar information
  • We can use semantic HTML elements to indicate the role of their content, as well as to give structure to the content of the webpage
  • Spaces between "header" elements and "body" elements are not visible to the user, but helps organize your code

GIT

  • git status: checks out what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git checkout "branch-name": opens an existing branch
  • git commands all start with "git"
  • git add -A: use this command to commit all files in the working branch (stages changes on local computer to prep for committing)
  • git commit: this command commit's files on our local device to get ready to push to the repo. It basically says you are done making changes on your local device, and are ready to push your changes.
  • git commit -m: the -m adds a message with our commit file (you should do this each time you commit changes)
  • git pull origin main: syncs our local branch with the main branch online to ensure you have the most updated version. Do this command before you push changes to the repo. Can do this after you commit changes to your local device.
  • git push origin "branch name": pushes changes from your local device to the online repo. The changes you committed in your working branch are being pushed for review on the repo.
